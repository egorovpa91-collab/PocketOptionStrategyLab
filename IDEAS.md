# IDEAS

## Архитектура

- Позже вынести список активов и интервал переключения в настройки GUI.

- Разделять Scanner, Candle Manager, Indicator Engine и Strategy Engine.
- Все вычисления запускать только от закрытых свечей.
- Использовать события для слабой связанности модулей.
- Не использовать TA-Lib как основу архитектуры.
- Предусмотреть повторное воспроизведение истории из SQLite.
- Позже вынести рыночные модели в независимый доменный пакет.

## Данные

- Проверять непрерывность M1 timestamp для каждой пары.
- Хранить источник свечи и качество данных.
- Добавить миграции схемы базы данных.
- Добавить диагностику локальных разрывов истории.
- Реализовать безопасное восстановление после аварийной остановки.

## Индикаторы

- Рассчитывать все необходимые индикаторы один раз на закрытую свечу.
- Хранить общий снимок индикаторов для повторного использования стратегиями.
- Поддерживать прогрев и минимальную длину истории.
- Версионировать формулы и параметры индикаторов.

## Стратегии

- Поддерживать тысячи комбинаций параметров.
- Оценивать стратегии отдельно по каждой паре.
- Анализировать устойчивость по времени суток и режиму рынка.
- Не использовать только WinRate: учитывать размер выборки и стабильность.
- Разделять исследовательские результаты и сигналы реального времени.

## GUI и отчёты

- Показывать состояние Scanner и Candle Manager.
- Выводить число закрытых свечей по каждой паре.
- Показывать разрывы истории и ошибки качества данных.
- Добавить тепловые карты эффективности.
- Экспортировать отчёты в Excel и PDF.
